Detailed Description

template<typename StateNameType, typename StateInterface>
class chestnut::statemachine::IStatemachine< StateNameType, StateInterface >

Base generic class for creating a statemachine.

Template Parameters
StateNameTypea type of states' name
StateInterfacea base (interface) class from which all states inherit

StateInterface type must inherit from IState to work properly. It can have any number of virtual methods that proper states should implement in some form

template<typename StateNameType , typename StateInterface >
void chestnut::statemachine::IStatemachine< StateNameType, StateInterface >::gotoState ( StateNameType  nextStateName)
virtual

Transitions directly to specified state, forgetting its previous state afterwards.

If state stack size is greater than 1, pops the state on the top of state stack and immediately pushes the specified state. State transition happens directly, without consideration of a state below the top. If state stack has only 1 element (the default state) the method acts like pushState(). If state with that name was not supplied in the constructor or the same state is on top of the stack, method does nothing.

Parameters
nextStateNamename of the state statemachine should transition to

template<typename StateNameType , typename StateInterface >
template<class DefaultState , class ... States>
void chestnut::statemachine::IStatemachine< StateNameType, StateInterface >::setupStates

Adds states of given types to the statemachine and initialize the default state.

States must inherit from StateInterface type. DefaultState type is the state type the statemachine is initialized with and will always stay on the bottom of state stack. When finished, method calls onEnter for the default state with its own name It is possible to pass in only default state type. This method should be called only once and always before calling any other method.

Template Parameters
DefaultStatetype of the default state for the statemachine
Statesrest of the states
